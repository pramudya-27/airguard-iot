Dokumentasi & Panduan Fitur

Tentang AIRGUARD

AIRGUARD adalah sistem monitoring kualitas udara, suhu, dan kelembapan ruangan berbasis Internet of Things (IoT). Alat ini dirancang untuk mendeteksi adanya gas berbahaya, asap, atau kebocoran gas secara real-time demi menjaga keamanan lingkungan sekitar.

Spesifikasi Perangkat Keras

NodeMCU ESP8266

Mikrokontroler berbasis WiFi yang bertugas membaca data sensor, memproses data logika lokal (seperti kontrol buzzer), dan mengirimkan data secara nirkabel via protokol MQTT.

Sensor DHT22

Sensor digital yang mengukur suhu (°C) dan kelembapan udara (%) dengan tingkat akurasi tinggi dan transmisi sinyal digital yang stabil.

Sensor MQ-135

Sensor kualitas udara gas elektrokimia yang sensitif terhadap gas berbahaya (Amonia, Benzena, Alkohol, Asap, CO2). Data diukur berdasarkan selisih (Delta) terhadap nilai acuan (Baseline).

Buzzer & Alarm

Aktuator suara sebagai indikator bahaya lokal. Buzzer akan berbunyi putus-putus cepat jika kualitas udara menyentuh kategori "BAHAYA".

Arsitektur Komunikasi Data

  1. Sensor -> ESP8266: ESP8266 membaca data DHT22 (suhu/kelembapan) dan MQ135 (Raw) setiap 2 detik.
  2. ESP8266 -> HiveMQ Cloud (MQTT): ESP8266 mempublikasikan payload JSON ke topik airguard/AIRGUARD-01/telemetry secara real-time via koneksi secure TLS (Port 8883).
  3. Flask Backend -> HiveMQ Cloud: Backend Python Flask berlangganan ke broker MQTT HiveMQ Cloud untuk menerima telemetry secara non-blocking di background thread.
  4. Penyimpanan SQLite: Backend menyaring data: menyajikan data instan ke memory RAM untuk Live Dashboard, namun hanya menyimpan data ke SQLite database setiap 30 detik untuk menghindari overload.
  5. Web Dashboard (SPA): Halaman web meminta data secara asinkronus (Fetch API) untuk menampilkan data real-time, grafik Chart.js, riwayat kompresi 1 menit, dan memicu kontrol perangkat.

Penjelasan Fitur Dashboard

Live Dashboard

Menampilkan widget interaktif suhu, kelembapan, status udara, dan grafik real-time yang diperbarui otomatis tiap 2 detik tanpa kedip.

Kontrol Jarak Jauh (ESP8266 Control Panel)

Panel kontrol nirkabel menggunakan protokol MQTT untuk mengirim perintah ke perangkat secara non-blocking:

  • Uji Buzzer: Membunyikan buzzer ESP8266 secara lokal selama 3 detik. Berguna untuk menguji integritas fisik/kabel aktuator buzzer tanpa harus menunggu paparan gas berbahaya.
  • Senyapkan 30 Detik: Menonaktifkan suara alarm buzzer selama 30 detik meskipun sensor MQ135 sedang mendeteksi kualitas udara dalam kondisi BAHAYA. Memberi waktu aman bagi pengguna untuk membersihkan udara kotor tanpa kebisingan.
  • Reset Alarm: Menghentikan bunyi alarm buzzer secara paksa dan mengembalikan indikator status bahaya perangkat (danger active) ke kondisi normal/aman.
  • Kalibrasi Baseline: Mengambil nilai rata-rata pembacaan sensor MQ135 (50 sampel) saat tombol ditekan untuk dijadikan nilai acuan (Baseline) udara bersih baru di memori perangkat. Memastikan pembacaan persentase gas tetap akurat secara dinamis.
Riwayat & Kompresi Data

Menampilkan data historis. Dilengkapi pilihan "Ringkasan 1 Menit" (data dirata-rata per menit agar rapi) dan "Semua Data" (data mentah per 30 detik), serta fitur ekspor CSV dan hapus data.

Pengaturan Fleksibel

Ambang batas (threshold) kualitas udara dan toggle "Auto Buzzer" dapat diatur dinamis dari web tanpa perlu menulis ulang (flash) kode mikrokontroler.
